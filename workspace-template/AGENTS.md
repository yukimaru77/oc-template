## Python 環境方針
Python の依存管理・実行環境には、可能な限り **uv** を優先して使う。

### 推奨コマンド
- `uv init`
- `uv add`
- `uv sync`
- `uv run`
- `uv lock`

### 原則
- 手動の `pip install` はできるだけ避ける
- 場当たり的な仮想環境作成は避ける
- グローバル Python への直接インストールは避ける
- Pythonのプロジェクト依存は **uv** で管理する
- 他の言語の場合でもそれらにあったスマートな管理方法を使用する
- プロジェクトがある程度完成したら誰の環境でもそれを再現できるように、markdown として残した環境構築手順を元にdockerfileの作成を行う
- ただし、dockerコンテナはあくまで配布用。普段の作業はこの環境で直接行う事(効率が悪いので)。プロジェクトができてから、dockerコンテナ化は行う
- あなたが司令塔になって、複数のコーディングエージェントもうまく活用する。この際評価は徹底的に行い少しの齟齬も許さず100％の出来になるまで直させる。

## 調査・Web 検索方針
外部情報、ドキュメント、関連 URL、背景知識が必要なときは、積極的に検索を行う。web_searchスキルで徹底的にキーワード検索行い検索結果を取得し、web_fetchで徹底的に本文を読み込みというお動作を反復的に何回も行うこと。かかる時間やコストは度外視し、品質を常に第一優先、常に調べてから行動する事。

### 検索を使う場面
- 外部情報が必要なとき
- 公式ドキュメントや仕様を探す必要があるとき
- 関連 URL を見つける必要があるとき
- 背景調査が必要なとき
- 少しでも情報が曖昧・不十分なとき

### 検索の進め方
- 1 回で済ませず、複数回検索して絞り込む
- その後fetchで本文を取得
- 別キーワードや言い換えを試す
- 結果が弱ければクエリを改善して再検索する
- 思い込みで進めず、必要に応じて繰り返し確認する。この分野はバージョンアップが激しいため容易に陳腐化します。

### 検索の役割分担
- 検索ツールは関連 URL の発見に使う
- 発見したページの本文確認は別ツールで行う
- 動的ページや JavaScript が多い場合は、それに適した取得手段を使う

### 調査の基本フロー
1. 検索で関連 URL を発見する
2. 結果を見て有望な候補を絞る
3. ページ本文や資料を取得して確認する
4. 不十分ならクエリを変えて再検索する

## 実装・検証方針
- 実装だけで終わらせず、可能な限り動作確認まで行う
- 再現手順を明確にする
- Docker / GPU / CUDA が必要な場合は、前提条件も含めて整理する
- 必要なら小さく段階を分けて検証し、失敗時は原因を切り分ける
- ベストエフォートではなく、可能な範囲で高品質になるまで反復する
