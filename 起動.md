基本は、**何もしなくていい**状態まで持っていけます。
今回の構成だと、再起動後に自動で戻るべきなのは

* **ホストの `oc-control-bot.service`**
* **Incus デーモン**
* **`boot.autostart=true` を入れた project コンテナ**
* **各コンテナ内の `openclaw.service`**

です。Incus の `boot.autostart` は、**daemon 起動時に instance を自動起動**する設定です。OpenClaw docs でも、**always-on / multi-user サーバーでは user service ではなく system service を使う**のが適しています。 ([Linux Containers][1])

なので、普段の運用はこう考えてください。

## ふだん

**電源を入れたら、そのまま Telegram で使う。**

---

## もし動かなかったら、確認はこの順だけ

### 1. ホストの管理 bot が生きているか

ホストで:

```bash
sudo systemctl status oc-control-bot --no-pager
```

### 2. Incus コンテナが起動しているか

```bash
incus list
```

`project-*` が `RUNNING` ならOK。
`boot.autostart=true` を入れてある instance は、Incus daemon 起動時に自動で上がる想定です。 ([Linux Containers][1])

### 3. ある project の OpenClaw が生きているか

```bash
incus exec project-name -- systemctl --system status openclaw --no-pager
incus exec project-name -- bash -lc 'source /etc/profile.d/openclaw-env.sh && openclaw gateway status --deep'
```

---

## もし手で復旧するなら

### 管理 bot を起こす

```bash
sudo systemctl restart oc-control-bot
```

### 全 project を起こす

```bash
incus start project-a
incus start project-b
```

### project の OpenClaw を起こす

```bash
incus exec project-a -- systemctl restart openclaw
```

---

## 一度だけ確認しておくと安心なこと

ホストでこれを見ておくと、今後かなり安心です。

```bash
sudo systemctl is-enabled oc-control-bot
incus config get project-a boot.autostart
incus exec project-a -- systemctl is-enabled openclaw
```

期待するのは:

* `oc-control-bot` → `enabled`
* `boot.autostart` → `true`
* `openclaw` → `enabled`

---

## かなり短く言うと

**理想状態では、電源ON後にあなたがやることは 0 です。**
もしダメでも、

1. `sudo systemctl restart oc-control-bot`
2. `incus list`
3. `incus exec project-x -- systemctl restart openclaw`

この3つだけ覚えておけば十分です。

必要なら次に、**再起動後の確認コマンドを1本の `check-everything.sh` にまとめる**ところまでやります。
